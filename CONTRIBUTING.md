# Contributing

Useful field notes are welcome. They do not need to sound academic or cover every possible payload.

A strong contribution does four things:

1. explains the security rule in plain English;
2. gives a focused way to investigate it;
3. points out likely false positives and stopping points;
4. keeps conclusions tied to repeatable evidence.

## Writing Style

Write like you are helping another hunter during a real test. Use short paragraphs, concrete examples, and direct language. Avoid filler, dramatic severity claims, and long lists with no explanation.

All wording and examples must be original. Never include credentials, personal information, private program material, or details from a target that cannot be shared.

## Before Opening a Pull Request

- Read your change once as a new hunter.
- Check that risky tests include clear limits.
- Make sure observations are not presented as proven impact.
- Run Markdown checks.
- Use the pull request checklist.
